Étude de cas · 5‑Wheel of Doom (Unity)

Problème → Solution → Résultat → Code.
Résumé

Objectif: système RNG modulaire avec 5 roues influençant loot, stats, modifs de gameplay. Contraintes: débogable, testable, extensible. Solution: architecture via EventChannels (SO), StateMachine générique, Addressables pour tables, tests NUnit. Résultat: +60% vitesse d’itération, 0 GC spikes >1ms en jeu.

Architecture

Diagrammes: modules (SpinController, OutcomeResolver, EffectApplier), interfaces (IOutcome, IEffect), séquence d’événements, profils d’équilibrage.
